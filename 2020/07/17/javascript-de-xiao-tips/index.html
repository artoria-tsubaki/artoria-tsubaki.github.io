2. Promise

  • 回调地狱：当需要保证程序的执行顺序的时候，只能通过回调函数调用，但过多的回调函数调用会造成回调地狱。

  • Promise 是一个构造函数，相当于一个容器，这个容器中存放一个异步任务。

  • Promise 的三种状态：

    • Pending
    • Resolved
    • Rejected

  • Promise 本身不是异步，但内部往往封装一个异步。

  • 调用 Promise 实例对象的 then 方法即是调用 Promise 构造函数中的形参中的函数。

a.txt
Hello aaa

b.txt
Hello bbb

var p1 = new Promise(function (resolve,reject) {
  fs.readFile('./data/a.txt','utf-8',function(err,data) {
    if(err) {
      reject(err)    //失败调用
    } else {
      resolve(data)    // 成功调用
    }
  })
})

var p2 = new Promise(function (resolve,reject) {
  fs.readFile('./data/b.txt','utf-8',function(err,data) {
    if(err) {
      reject(err)    //失败调用
    } else {
      resolve(data)    // 成功调用
    }
  })
})

p1.then(function(data) {
  console.log(data) // 成功了即调用 resolve 这个传递函数
  // 返回 Hello aaa
  return p2
},function(err) {
  // 失败了即调用 reject 这个传递函数
  console.log('读取文件失败了',err)
})
.then(function(data) { // 该方法会作为 p2 的 resolve
  console.log(data) // 返回 Hello bbb
})

5.apply、call、bind 的用法

  • 三者都用来重新定义 this 这个对象
var obj1 = {
  name: '张三',
  objAge: this.age,
  myFun: function () {
    console.log(this.name + '年龄' + this.age)
  }
}

var obj2 = {
  name: '张三',
  objAge: this.age,
  myFun: function () {
    console.log(this.name + '年龄' + this.age + '，来自' + fm + '去往' + t)
  }
}

var other = {
  name: '老王',
  age: 18
}
  • 类型的区别：bind 方法返回的是一个全新的函数，需要调用才会执行，而 apply 和 call 方法都是对方法的调用
obj1.myFun.call(other) //老王年龄18
obj1.myFun.apply(other) //老王年龄18
obj1.myFun.bind(other)() //老王年龄18
  • 传参的区别：第一个参数都是 this 的指向对象，第二个参数的区别如下
    • call 的参数是直接放进去的，参数都用逗号分隔
    • apply 的所有参数都需要放在一个数组中
    • bind 的参数是直接放进去的，参数都用逗号分隔
obj2.myFun.call(other, '成都', '上海')
// 老王年龄18，来自成都去往上海
obj2.myFun.apply(other, ['成都', '上海'])
// 老王年龄18，来自成都去往上海
obj2.myFun.bind(other, '成都', '上海')()
// 老王年龄18，来自成都去往上海
obj2.myFun.bind(other, ['成都', '上海'])()
//老王年龄18，来自成都上海去往undefined

5.1 方法借用

  • 借用 Math 的 max 和 min 方法求数组的最大最小值
/*
    不能使用 call 方法，参数会被当成一个数组。
    使用 apply 方法，参数是一个个数字。
*/
var maxOfArr = Math.max.apply(null, [3, 5, 7, 8, 4]) //8
var maxOfArr = Math.max.call(null, [3, 5, 7, 8, 4]) // NaN

var minOfArr = Math.min.apply(null, [3, 5, 7, 8, 4]) //3

10. localStorage 操作

  • localStorage 用于在浏览器端持久化保存键值对 数据
  • 写入用户硬盘，从硬盘中写入读出数据
  • localStorage 特点
    • 大小限制：5M(chrome)，更大数据可以使用 浏览器本地数据库（indexDB 或 webSql）
    • 受同源访问限制，不能跨域访问
    • 在浏览器隐私模式下无法使用
    • 因为在本地保存，不会发送数据，网络爬虫无法抓取
    • 只能存放字符串（JSON 字符串）
  • localStorage 基本语法
    • setItem(‘key’,’value’)
    • getItem(‘key’)
    • removeItem(‘key’)
    • clear()

11. 作用域与闭包问题

11.1 函数的作用域

  • 每次函数被重新调用，都会创建一个 新的内存空间和地址 。
  • 延伸生命周期：当函数被外部引用（使用）时，通过这个引用调用函数不会创建新的内存空间，示例：
// 每次被调用后当前内存空间被销毁
function test() {
  var n = 1
  function hd() {
    console.log(++n)
  }
}

test() // 2
test() // 2

// 被外界引用不会被销毁
function test() {
  var n = 1
  return function hd() {
    console.log(++n)
  }
}

var a = test()
a() // 2
a() // 3

// 多层嵌套 内部没有被引用，所以调用会重新创建新的空间
function test() {
  var n = 1
  return function hd() {
    var m = 1
    function ss() {
      console.log(++m)
    }
  }
}

var b = test()
b() // 2
b() // 2

function test() {
  var n = 1
  return function hd() {
    var m = 1
    return function ss() {
      console.log(++m)
    }
  }
}

var b = test()()
b() // 2
b() // 3

11.2 块级作用域

  • var 没有块级作用域的特性，在块级作用域声明也会提升到全局
function test() {
  for (var i = 1; i <= 3; i++) {
    setTimeout(function () {
      console.log(i)
    }, 1000)
  }
}

test() // 一秒钟后直接输出 4,4,4
  • let 和 const 有块级作用域，在这个块中是独立的
function test2() {
  for (let i = 1; i <= 3; i++) {
    setTimeout(function () {
      console.log(i)
    }, 1000)
  }
}

test2() // 一秒钟后直接输出 1,2,3

function test3() {
  for (let i = 1; i <= 3; i++) {
    setTimeout(function () {
      var j = 2
      console.log(j++)
    }, 1000)
  }
}

test3() // 一秒钟后输出 2,2,2
  • 模拟出 var 的伪块级作用域
function test4() {
  for (var i = 1; i <= 3; i++) {
    ;(function (i) {
      setTimeout(function () {
        console.log(i)
      }, 1000)
    })(i)
  }
}

test4() // 一秒钟后输出 1,2,3
// 解析：利用立即函数 将 window 下的 i 作为参数传入到函数体中，每一个  i 都在一个独立的作用域中

11.3 闭包

  • 闭包指的是： 能够访问另一个函数作用域变量的函数
// 利用闭包根据字段排序商品
var cart = [
  { name: 'iphone', click: 123, price: 8000 },
  { name: 'ipad', click: 80, price: 4000 },
  { name: 'icoo', click: 150, price: 10000 }
]

function order(filed) {
  return cart.sort(function (a, b) {
    return a[filed] > b[filed] ? 1 : -1
  })
}

console.log(order('price')) // 按价格升序排序

// 利用闭包特性行，内部 cart 的 sort 方法可以访问外部函数的变量 filed
  • 内存泄漏问题：
    • 当函数内部被使用时，其中的数据一直被占用不会被释放，当我们仅需要一部分数据时，可以把不需要的赋值为 null，来减少内存使用
<button desc="artoria">artoria</button>
<button desc="saber">saber</button>

<script>
  var btns = document.querySelectorAll('button')
  btns.forEach(function (item) {
    var desc = item.getAttribute('desc')
    item.addEventListener('click', function () {
      console.log(desc) // artoria / saber
    })
    item = null
    console.log(item) // null
  })
</script>

14. 任务执行顺序

14.1 微任务

  • 定义：由语言标准提供的叫微任务。
  • 语言标准：即是 ECMAScript。
  • 类型：ES6 提供的 promise.then , process.nextTick

14.2 宏任务

  • 定义：宿主环境提供的叫宏任务。
  • 类型：
    • 所有内建或自定义函数。
    • global 或 window 这个全局对象的属性或方法。
    • setTimeout 、 setInterval
  • 宿主环境：global 是 JavaScript 运行时所在宿主环境提供的全局对象。在浏览器中，没有实现 global 对象，而是通过 window 对象来指向 global，代替 global 称为全局对象。在 node 中，用到的全都是 JavaScript 的原生功能。

微任务早于宏任务执行，微任务和宏任务都是异步任务

14.3 同步任务

  • 定义：在主线程上排队执行的任务，只有前面的任务执行完毕才能执行后一个任务。
  • 例子：promise 实例执行的代码，全局范围下的 console.log

14.4 异步任务

  • 定义：不进入主线程，而是进入 任务队列 的任务，只有主线程内的任务都执行完毕，才会去任务队列读取相应的函数，进入主线程执行。

  arr.reduce((promise, curr) => {
    return promise.then((_) => {
      return curr()
    })
  }, Promise.resolve())
}
