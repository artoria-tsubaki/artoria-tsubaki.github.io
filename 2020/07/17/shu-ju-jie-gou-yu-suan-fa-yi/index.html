数据结构与算法

1.What

2. 数组

  • 数组是一种 线性结构
  • 数组的优点：
    • 通过下标查找数据十分方便。
  • 数组的缺点：
    • 创建需要申请一段连续的内存空间，大小是固定的，可能会需要扩容（耗时耗力）
    • 添加和删除元素的效率低，可能会需要进行大量的元素的

3. 栈

  • 栈是一种受限的线性结构，只能从表的一端添加或删除元素。这一端称为栈顶 ，另一端称为栈底。
  • 栈的特点是后进先出（LIFO）
  • 函数调用栈
    • A 调用 B，B 中又调用 C，C 中又调用 D
    • 在执行过程中，会先将 A 压入栈，A 没有执行完，所以不会出栈
    • 在 A 的执行过程中调用了 B ，所以 B 压入栈，这时 B 在栈顶，A 在栈底
    • B 调用了 C ，所以 C 会压栈，并且在栈顶，而 C 调用了 D，D 会压入栈顶
    • 当前栈的顺序是 A –> B –> C –> D
    • 出栈的顺序为 D –> C –> B –> A
  • 递归调用没有跳出条件的时候会造成 栈溢出 的现象。
  • 栈的实现
// 通过 类 来实现 栈
class Stack {
  // 通过 数组 来实现 栈中数据的存放
  items = []

  // 栈的相关操作
  // 1. 将元素压入栈
  push(element) {
    this.items.push(element)
  }

  // 2. 从栈中取出元素
  pop() {
    return this.items.pop()
  }

  // 3. 查看一下栈顶元素
  peek() {
    return this.items[this.items.length - 1]
  }

  // 4. 判断栈是否为空
  isEmpty() {
    return this.items.length === 0
  }

  // 5.获取栈中元素的个数
  size() {
    return this.items.length
  }

  // 6. toString 方法
  toString() {
    return this.items.join(' ')
  }
}
  • 对栈的使用： 实现 NumberObject.toString(2) 十进制转二进制
function dec2bin(decNumber) {
  var s = new Stack()

  // decNumber 为 0 时退出循环
  while (decNumber > 0) {
    // 在栈中存储 十进制 模二 的余数
    s.push(decNumber % 2)
    // 将 decNumber 除二 赋予自身
    decNumber = Math.floor(decNumber / 2)
  }

  var binNumber = ''
  // s 中还有数据都赋值给 binNumber
  while (!s.isEmpty()) {
    // 字符串拼接
    binNumber += s.pop()
  }
  return binNumber * 1
}

4. 队列

  • 队列也是一种受限的线性结构，只允许在表的 前端 进行删除操作，表的 后端 进行插入操作

  • 队列的特点是先进先出（FIFO）

  • 队列的应用：

    • 打印队列：优先放入的文档，优先被取出。
    • 线程队列：线程队列会依照次序来实现线程。

  • 队列的实现：

class Queue {
  // 通过 数组 来实现 队列中数据的存放
  items = []

  // 队列中的相关操作
  // 1. 实现 队列 的添加操作
  enqueue(element) {
    this.items.push(element)
  }

  // 2. 实现 队列 的删除操作
  delqueue() {
    return this.items.shift()
  }

  // 3. 查看 队列的 最前面的元素
  front() {
    return this.items[0]
  }

  // 4. 判断 队列 是否为空
  isEmpty() {
    return this.items.length === 0
  }

  // 5. 队列的 大小
  size() {
    return this.items.length
  }

  // 6. 重写 toString 方法
  toString() {
    // return this.items.join(' ')
    var str = ''
    for (var i = 0; i < this.items.length; i++) {
      str += this.items[i] + ' '
    }
    return str
  }
}
  • 击鼓传花的实现
    • 规则：几个朋友一起玩一个游戏，围成一圈，开始数数，数到某个数字的人自动淘汰。最后剩下的这个人会获得胜利，请问最后剩下的是原来在哪一个位置的人。
function passGame(nameList, num) {
  var q = new Queue()
  // 1. 将 玩家存入 队列
  for (var i = 0; i < nameList.length; i++) {
    q.enqueue(nameList[i])
  }

  // 3. 直到 队列 的长度 等于 1 停止操作
  while (q.size() > 1) {
    // 2.1 小于 num 的数 取出后存入 尾部
    for (var a = 0; a < num - 1; a++) {
      q.enqueue(q.delqueue())
    }
    // 2.2 等于 num 的数 直接删除
    q.delqueue()
  }
  console.log(q.size())
  var endName = q.front()
  console.log('最终的人是：' + endName)
  return nameList.indexOf(endName)
}

4.1 优先级队列

  • 优先级队列的特点：在插入每一个元素的时候回考虑数据的优先级，和其他数据优先级进行比较，在比较完成后，可以得出这个元素只在队列中正确的位置。
  • 需要考虑的问题：每个元素不再只是一个数据，而且包含数据的优先级。
  • 生活中的应用：机场登机顺序，急诊科的处理，通过优先级队列重新排序队列中的顺序
  • 优先级队列的实现：
function priorityQueue() {
  // 定义 一个数组 接收 数据对象
  this.items = []

  // 定义 一个数据对象类
  function queueElement(name, priority) {
    this.name = name
    this.priority = priority
  }

  // 实现 队列 插入数据的功能
  priorityQueue.prototype.enqueue = function (name, priority) {
    var newElem = new queueElement(name, priority)
    // 数组中没有数据 直接存放
    if (this.items.length === 0) {
      this.items.push(newElem)
    } else {
      // 判断插入的数 是否 都大于 数组中的数，表示是否 已插入
      var isAdded = false
      for (var i = 0; i < this.items.length; i++) {
        if (this.items[i].priority > newElem.priority) {
          // 插入
          this.items.splice(i, 0, newElem)
          // 说明 已插入
          isAdded = true
          break
        }
      }
      // 插入的数 大于 所有 数组中的数
      if (!isAdded) {
        // 直接插入
        this.items.push(newElem)
      }
    }
  }

  // 实现 队列 移除数据的功能
  priorityQueue.prototype.dequeue = function () {
    this.items.shift()
  }
}

5. 链表

  • 功能：存储多个元素
  • 特点：
    • 链表中的元素在内存中不必是连续的空间。
    • 每个元素由一个存储元素本身的节点和一个指向下一个元素的引用（指针）组成
  • 相对于数组的优点：
    • 内存空间不是必须连续的，可以实现灵活的内存动态管理
    • 链表创建的时候不必确定大小，可以无限的延伸下去
    • 链表在插入和删除数据时，时间复杂度能达到 O(1)，效率高很多。
  • 相对于数组的缺点：
    • 访问任何位置的元素都需要从头开始访问
    • 无法通过下标直接访问元素
  • 什么是链表
    • 类似于 火车 ，有一个火车头，火车头会连接一个节点，节点上有乘客（类似于数据），并且这个节点会连接下一个节点，一次类推。

5.1 单向链表

  • 只能从头遍历到尾 或 从尾部遍历到头部

  • 实现的原理是上一个链表中有一个指向下一个的引用

  • 缺点：

    • 可以轻松的到达下一个节点，但是回到前一个节点是很难的。

  • 单向链表的实现

function LinkedList() {
  // 内部类 存储数据的类
  function DataElement(data) {
    this.data = data
    this.next = null
  }

  // 内部属性
  // 默认 head 指向 null
  this.head = null
  this.length = 0

  // 实现 添加方法
  LinkedList.prototype.append = function (data) {
    // 1. 创建 数据类
    var newNode = new DataElement(data)

    // 2. 判断添加的 是否是 第一个节点
    if (this.length === 0) {
      this.head = newNode
    } else {
      // 2.1获取 当前第一个节点
      var currentNode = this.head
      // 2.2 获取 最后一个节点
      while (currentNode.next) {
        currentNode = currentNode.next
      }
      // 2.3 在当前最后一节节点 追加
      currentNode.next = newNode
    }
    this.length += 1
  }

  // 实现 toString 方法
  LinkedList.prototype.toString = function () {
    var resultStr = ''
    var current = this.head
    while (current) {
      resultStr += current.data + ' '
      current = current.next
    }
    return resultStr
  }

  // 实现 插入方法
  LinkedList.prototype.insert = function (position, data) {
    // 创建 数据类
    var newNode = new DataElement(data)
    // 当前 读取到的 数据类
    var index = 0
    // 当前 未插入前的 第一个数据类
    var current = this.head

    // 1. 判断 position 是否越界
    if (position < 0 || position > this.length) return false
    // 2. 在 表头 插入
    if (position === 0) {
      // 第一个 数据类
      this.head = newNode
      // 插入的 数据类 的 next 指向原来的 第一个数据类
      newNode.next = current
    } else {
      var prev = null
      // 3. 在 表中 插入
      while (index++ < position) {
        prev = current
        current = current.next
      }
      prev.next = newNode
      newNode.next = current
    }
    // 4.长度加一
    this.length += 1
  }

  // 实现 get 获取数据 方法
  LinkedList.prototype.get = function (position) {
    if (position < 0 || position > this.length - 1) return false
    var index = 0
    var current = this.head
    while (index++ < position) {
      current = current.next
    }
    return current.data
  }

  // 实现 indexOf 方法
  LinkedList.prototype.indexOf = function (data) {
    var index = 0
    var current = this.head
    while (current) {
      if (current.data === data) {
        return index
      } else {
        current = current.next
        index++
      }
    }
    if (current == null) {
      return -1
    }
  }

  // 实现 update 方法
  LinkedList.prototype.update = function (position, newData) {
    if (position < 0 || position > this.length - 1) return false
    var index = 0
    var current = this.head
    while (index++ < position) {
      current = current.next
    }
    current.data = newData
    return true
  }

  // 实现 removeAt 方法
  LinkedList.prototype.removeAt = function (position) {
    if (position < 0 || position > this.length - 1) return null
    var current = this.head
    if (position === 0) {
      this.head = this.head.next
    } else {
      var prev = null
      var index = 0
      while (index++ < position) {
        prev = current
        current = current.next
      }
      prev.next = current.next
    }
    this.length -= 1
    return current.data
  }

  // 实现 remove 方法
  LinkedList.prototype.remove = function (data) {
    var index = this.indexOf(data)

    return this.removeAt(index)
  }

  // 实现 isEmpty 方法
  LinkedList.prototype.isEmpty = function () {
    return this.length === 0
  }

  // 实现 size 方法
  LinkedList.prototype.size = function () {
    return this.length
  }
}

5.2 双向链表

  • 既可以从头遍历到尾，也可以从尾遍历到头
  • 实现原理：一个节点即有向前连接的引用，向后连接的引用
  • 缺点：
    • 插入或删除需要处理 四个引用
    • 相比于单向链表，占用内存空间更大

瑕不掩瑜，这些缺点对于操作的方便性是微不足道的。

  • 双向链表的实现：
function DoulyLinkedList() {
  // 定义 内部类 封装数据
  function DataElement(data) {
    this.data = data
    this.next = null
    this.prev = null
  }

  // 内部属性
  this.length = 0
  this.head = null
  this.tail = null

  // append 尾部追加方法 的实现
  DoulyLinkedList.prototype.append = function (data) {
    var newElem = new DataElement(data)
    // 判断 双向链表 是否为空
    if (this.length === 0) {
      this.head = newElem
      this.tail = newElem
    } else {
      this.tail.next = newElem
      newElem.prev = this.tail
      this.tail = newElem
    }
    // 长度加一
    this.length += 1
  }

  // forwardString 正向遍历
  DoulyLinkedList.prototype.forwardString = function () {
    var index = 0
    var current = this.head
    var str = ''
    while (index++ < this.length) {
      str += current.data + ' '
      current = current.next
    }
    return str
  }

  // backwardString 倒向遍历
  DoulyLinkedList.prototype.backwardString = function () {
    var index = 0
    var current = this.tail
    var str = ''
    while (index++ < this.length) {
      str += current.data + ' '
      current = current.prev
    }
    return str
  }

  // toString 方法 的实现
  DoulyLinkedList.prototype.toString = function () {
    return this.forwardString()
  }

  // insert 插入方法 的实现
  DoulyLinkedList.prototype.insert = function (position, data) {
    // 创建数据类
    var newElem = new DataElement(data)

    // 1. 越界判断
    if (position < 0 || position > this.length) return false

    // 2. 插入的实现
    if (this.length === 0) {
      // 2.1 链表内没有数据直接插入
      this.head = newElem
      this.tail = newElem
    } else {
      // 2.2 插入第一个位置
      if (position === 0) {
        this.head.prev = newElem
        newElem.next = this.head
        this.head = newElem
      } else if (position === this.length) {
        // 2.3 插入最后一个位置
        this.tail.next = newElem
        newElem.prev = this.tail
        this.tail = newElem
      } else {
        // 2.4 插入中间的位置
        var index = 0
        var current = this.head
        while (index++ < position) {
          current = current.next
        }
        current.prev.next = newElem
        newElem.prev = current.prev
        newElem.next = current
        current.prev = newElem
      }
    }
    // 3. 长度加一
    this.length += 1
  }

  // get 方法 的实现
  DoulyLinkedList.prototype.get = function (position) {
    if (position < 0 || position > this.length - 1) return null
    // 1. 判断 position 在链表的位置
    // 1.1 在链表的前半部分，用正向查询
    if (this.length / 2 >= position) {
      var index = 0
      var current = this.head
      while (index++ < position) {
        current = current.next
      }
      return current.data
    } else {
      // 1.2 在链表的后半部分，用反向查询
      var index = 0
      var current = this.tail
      while (index++ < this.length - 1 - position) {
        current = current.prev
      }
      return current.data
    }
  }

  // indexOf 方法 的实现
  DoulyLinkedList.prototype.indexOf = function (data) {
    var index = 0
    var current = this.head
    while (current) {
      if (current.data === data) {
        return index
      } else {
        current = current.next
        index++
      }
    }
    return -1
  }

  // update 方法 的实现
  DoulyLinkedList.prototype.update = function (position, data) {
    if (position < 0 || position > this.length - 1) return false
    var index = 0
    // 判断 position 的位置 在链表中的位置 来决定正向 反向查询

    var current = this.head
    while (index++ < position) {
      current = current.next
    }
    current.data = data
    return true
  }

  // removeAt 方法 的实现
  DoulyLinkedList.prototype.removeAt = function (position) {
    if (position < 0 || position > this.length - 1) return false
    // 1.1 移除 头部第一个数据
    if (position === 0) {
      // 1.2 当链表只有一个数据
      if (this.length === 1) {
        this.head = null
        this.tail = null
      } else {
        // 1.3 当链表有多个数据
        this.head = this.head.next
        this.head.prev = null
      }
    } else if (position === this.length - 1) {
      // 2.1 移除 尾部的第一个数据
      this.tail = this.tail.prev
      this.tail.next = null
    } else {
      // 3. 移除 中间的数据
      var index = 0
      // 3.1 判断 删除的元素位置 在链表 的对应位置
      if (this.length / 2 >= position) {
        var current = this.head
        while (index++ < position) {
          current = current.next
        }
        current.next.prev = current.prev
        current.prev.next = current.next
      } else {
        var current = this.tail
        while (index++ < this.length - 1 - position) {
          current = current.prev
        }
        current.next.prev = current.prev
        current.prev.next = current.next
      }
    }
    this.length -= 1
    return true
  }

  // remove 方法 的实现
  DoulyLinkedList.prototype.remove = function (data) {
    var position = this.indexOf(data)
    return this.removeAt(position)
  }

  // isEmpty 方法 的实现
  DoulyLinkedList.prototype.isEmpty = function () {
    return this.length === 0
  }

  // size 方法 的实现
  DoulyLinkedList.prototype.size = function () {
    return this.length
  }
}

6. 集合

  • 特点：无序，不能重复的元素构成
  • ES6 封装的 Set 类 即是 集合类
  • 集合类的实现
class mySet {
  // 创建 一个对象 来接收 数据
  items = {}

  // 实现 add 方法 向集合添加一个新的项
  add(value) {
    // 1. 判断是否存在这个值
    if (!this.has(value)) {
      this.items[value] = value
      return true
    }
    return false
  }

  // 实现 remove 方法 从集合移除一个值
  remove(value) {
    if (this.has(value)) {
      delete this.items[value]
      return true
    }
    return false
  }

  // 实现 has 方法 判断值是否在集合中 返回值为 布尔
  has(value) {
    return this.items.hasOwnProperty(value)
  }

  // 实现 clear 方法 移除集合中所有项
  clear() {
    this.items = {}
  }

  // 实现 size 方法
  size() {
    return Object.keys(this.items).length
  }

  // 实现 values 方法 返回一个包含集合所有值的数组
  values() {
    return Object.values(this.items)
  }
}

7. 字典

  • 特点：
    • 存储的是键值对
    • 一个键对应一个值
    • 键是不允许重复的
  • 在 JavaScript 中 字典和对象之间基本可以相互替代
  • 在很多编程语言（如 java）中，对象通常实在编译期就确定下来的结构，不可以动态的添加或删除，字典通常会使用类似 哈希表 的数据结构去实现一种动态的添加数据的结构

8. 哈希表

  • 哈希表是一种根据 键值对而直接访问的数据结构
  • 给定表 M，存在函数 f(key)，对任意给定的关键字值 key，代入函数后或能得到包含该关键字的记录在表中的地址，则称表 M 为哈希表，函数为哈希函数
  • 哈希表的优点：
    • 提供非常快速的 插入-删除-查找 操作
    • 无论多少数据，插入语和删除值需要非常接近常量的时间，即 O(1) 的时间级
    • 查找速度比数还要快，基本可以瞬间找到想要的元素。
  • 哈希表的缺点：
    • 哈希表中的数据是没有顺序的，所以不能以一种固定的方式（比如从小到大）来遍历其中的额元素。
    • 一般而言，哈希表中的 key 是不允许重复的，不能放置相同的 key 保存不同的元素。

8.1 冲突

  • 通过哈希函数得到一个数所在位置已经被其他数存储了，这种现象称为 冲突。
  • 解决方案：
    • 链地址法
    • 开放地址法

8.2 链地址法

  • 每个数组单元中存储的不再是单个数据，而是一个链条（数组或链表）
  • 一旦发现重复，将重复的元素插入到链表的首端或者末端即可
  • 当查询时，先根据哈希化后的下标值找到对应的位置，再取出链表，依次查询要寻找的数据
  • 链条的选择根据业务需要来决定。当需要插入到首位时（新插入的数据用于取出的可能性比较大），有限选择 链表。

8.3 开放地址法

  • 寻找空白的单元格来添加重复的数据。

  • 探索空白位置的三种方式：

    • 线性探测

      ​ 往后 +1 插入 \ 查找 \ 删除，查找的空位置就停止，删除则将该位置的内容置 -1

    • 二次探测

      ​ 对步长做了优化, 比如从下标值 x 开始, x+1², x+2², x+3²，可以一次性探测比较长的距离

    • 再哈希法

      ​ 把关键字再用另外一个哈希函数，再做一次哈希化，用这次的结果作为步长

一般而言，链地址法相对于开放地址法来说效率好，使用链地址法的情况也比较多

8.4 哈希表的实现

function HashTable() {
  // 属性
  this.storage = []
  // 存放的 数据 个数
  this.count = 0
  this.limit = 7

  // 方法

  // 设计 哈希函数
  // 1. 将 字符串 转换成 比较大的数字：hashCode
  // 2. 将 大的数字 hashCode 压缩到 数组范围（大小）内
  HashTable.prototype.hashFunc = function (str, size) {
    // 1. 定义 hashCode 变量
    var hashCode = 0
    // 2. 霍纳算法 计算 hashCode 的值
    // 2.1 cats => Unicode 编码
    for (var i = 0; i < str.length; i++) {
      hashCode = 37 * hashCode + str.charCodeAt(i)
    }

    // 3. 取余操作
    var index = hashCode % size
    return index
  }

  // 插入 & 修改 操作 的实现
  HashTable.prototype.put = function (key, value) {
    // 1. 通过 哈希化 index 获取 对应的 index
    var index = this.hashFunc(key, this.limit)

    // 2. 根据 index 创建 对应的 bucket
    var bucket = this.storage[index]

    // 3. 若 bucket 为 undefined 则 需要创建
    if (bucket == undefined) {
      bucket = []
      this.storage[index] = bucket
    }
    // 4. 判断 是否是 修改操作
    var override = false
    for (var i = 0; i < bucket; i++) {
      var tuple = bucket[i]
      if (tuple[0] == key) {
        tuple[1] = value
        override = true
      }
    }

    // 5. 进行 插入操作
    if (!override) {
      bucket.push([key, value])
      this.count += 1
      // 数组扩容
      if (this.count / this.limit > 0.75) {
        var newPrime = this.getPrime(this.limit * 2)
        this.resize(newPrime)
      }
    }
  }

  // 获取 操作 的实现
  HashTable.prototype.get = function (key) {
    // 1. 根据 key 获取 对应的 index
    var index = this.hashFunc(key, this.limit)

    // 2. 根据 index 获取 对应的 bucket
    var bucket = this.storage[index]
    console.log(bucket)
    // 3. 判断 bucket 是否为 undefined
    if (bucket == undefined) {
      return null
    } else {
      // 4. 有 bucket 那么就进行 线性查询
      for (var i = 0; i < bucket.length; i++) {
        var tuple = bucket[i]
        if (tuple[0] === key) {
          return tuple[1]
        }
      }
    }
    // 5. 依然没有找到，那么返回 null
    return null
  }

  // 删除 操作 的实现
  HashTable.prototype.remove = function (key) {
    // 1. 通过 key 获取 对应的 index
    var index = this.hashFunc(key, this.limit)

    // 2. 通过 index 获取 对应的 bucket
    var bucket = this.storage[index]

    // 3. 如果 bucket 不存在 返回 null
    if (bucket == undefined) {
      return null
    }
    // 4. 线性查找 bucket 中的元素，删除对应的值
    for (var i = 0; i < bucket.length; i++) {
      var tuple = bucket[i]
      if (tuple[0] === key) {
        bucket.splice(i, 1)
        this.count -= 1

        if (this.limit > 7 && this.count / this.limit < 0.25) {
          var newPrime = this.getPrime(Math.floor(this.limit / 2))
          this.resize(newPrime)
        }

        return tuple[1]
      }
    }
    // 5. 如果依然没找到 返回 null
    return null
  }

  // 判断 是否为空 的函数 实现
  HashTable.prototype.isEmpty = function () {
    return this.count === 0
  }
}

8.5 哈希表扩容的思想

  • 为什么需要扩容
    • 链地址法，loadFactor 可以大于 1， 所以这个哈希表可以无限制的插入新数据
    • 随着数据量的增多，每一个 index 对应的 bucket 会越来越长，也就导致效率降低。
  • 如何进行扩容
    • 一般可以增大两倍然后向上取最近的质数
    • 所有数据都要同时进行修改
    • 虽然耗时但是是必须的
  • 在什么情况下扩容
    • 当 loadFactor > 0.75 时进行扩容

loadFactor 表示哈希表中元素的填满程度

  • 实现：
// 哈希表 扩容
HashTable.prototype.resize = function (newLimit) {
  // 1. 保存 旧的数组内容
  var oldStorage = this.storage

  // 2. 重置属性
  this.storage = []
  this.count = 0
  this.limit = newLimit

  // 3. 遍历 旧数组中的所有数据项，并且 重新插入到 哈希表中
  for (var i = 0; i < oldStorage.length; i++) {
    var bucket = oldStorage[i]
    // 3.1 bucket 为 null，说明这里没有数据
    if (bucket === null) {
      continue
    }
    // 3.2 bucket 中有数据，那么将里边的数据 重新 哈希化赋值
    for (var j = 0; j < bucket.length; j++) {
      var tuple = bucket[j]
      this.put(tuple[0], tuple[1])
    }
  }
}

// 判断 质数 的函数
HashTable.prototype.isPrime = function (num) {
  var temp = parseInt(Math.sqrt(num))

  for (var i = 2; i <= temp; i++) {
    if (num % i == 0) {
      return false
    }
  }
  return true
}

// 向上取 最近的质数
HashTable.prototype.getPrime = function (num) {
  while (!isPrime(num)) {
    num++
  }
  return num
}
