Vue.js

1.1 框架模式

  • MVC
  • MVP
  • MVVM

1.2 Vue.js 是什么

  • Vue.js 不是一个框架，它只是聚焦视图层，是一个构建数据驱动的 Web 界面的库。通过简单的 API 提供了高效的数据绑定和灵活的组件系统。
  • Vue.js 的特性
    • 确实轻量
    • 数据绑定
    • 指令
    • 插件化

2.1 插值表达式

2.1.1 插值

  • 文本插值是最基本的形式，使用双大括号 { {} } 来实现。
<span>Text: { {text} }</span>
  • 标签 { {text} } 将会被相应的数据对象 text 属性的值替换掉，当 text 的值改变的时候，文本的值也会相应发生改变。
  • 当只需渲染一次数据，后续数据变化不再关心可以通过 * 来实现
<span>Text: { {*text} }</span>
  • 双大括号标签会把里边的值全部当成字符串来处理，如果是 HTML 片段，则可以使用三个大括号来绑定。
<div> { {{logo} }}</div>
logo: '<span>DDEF</span>'
  • 双大括号标签还可放在 HTML 标签内。
<li data-id="{ {id} }"></li>

2.1.2 表达式

  • 双大括号标签也支持表达式形式的值，表达式可以由 JavaScript 表达式和过滤器构成。
<!-- JS 表达式 -->
{ { cents/100 } } { { true? 1 : 0 } } { { example.split(",") } }

<!-- 过滤器 -->
{ { example | toUpperCase } } { {example | filterA | filterB} }

2.2 分隔符

通过配置目录文件src/config.js中的 vue.config.delimiters 来修改默认的文本插值的分隔符

3.1 指令

  • 指令是特殊的带有前缀 v- 的特性。
  • 指令的值限定为绑定表达式。
  • 指令的功能就是当前表达式改变时把某些特殊的行为应用到 DOM 上。

3.2 内部指令

3.2.1 v-if

  • v-if 指令可以完全根据表达式的值在 DOM 中生成或移除一个元素。
  • 如果 v-if 表达式赋值为 false，那么对应的元素就会从 DOM 中移除。
  • 如果 v-if 表达式赋值为 true，那么对应的元素的一个克隆将被重新插入 DOM 中。
  • 下例代码中的 p 元素即从 DOM 中移除。
<!-- 切换单个元素 -->
<body class="native">
  <div id="example">
    <p v-if="greeting">Hello</p>
  </div>
</body>

<script>
  var exampleVM2 = new Vue({
    el: '#example',
    data: {
      greeting: false
    }
  })
</script>

<!-- 切换多个元素，可以用 template 标签包装元素 -->
<template v-if="ok">
  <h1>title</h1>
  <p>paragraph1</p>
  <p>paragraph2</p>
</template>

3.2.2 v-show

  • 根据表达式的值来显示或隐藏 HTML 元素
  • 当 v-show 赋值为 false 时，元素将被隐藏。查看 DOM 时，元素上多了一个内联样式 style="display:none"。
  • 下例代码的 p 元素被隐藏,其样式为 display:none。
<div id="app">
  <p v-show="greeting">Hello!</p>
</div>

var vm = new Vue({ el: '#app', data: { message: '', greeting: false, } })
  • v-show 不支持 < template > 语法
  • v-if 是个真实的条件渲染，它是惰性的——如果初始渲染的条件是 false，则什么也不做，在条件第一次变为 true 时才会开始局部编译（编译会被缓存）；v-show 元素始终被编译并保留，只是简单地基于 CSS 切换。
  • 一般来说，v-if 有更高的切换消耗，而 v-show 有更高的渲染消耗。因此当需要频繁地切换，使用 v-show 更好；在运行时条件不大可能改变，使用 v-if 更好。

3.2.3 v-else

  • v-else 就是 JavaScript 中 else 的意思，它必须跟着 v-if 或者 v-show，从当 else 功能。
<div id="app">
  <p v-if="ok">我是对的！</p>
  <p v-else>我是错的！</p>
</div>

<script>
  var vm = new Vue({
    el: '#app',
    data: {
      ok: false
    }
  })
</script>

<!-- 当我们使用 v-show的时候，可使用另一个 v-show 来替换 v-else -->
<p v-show="ok">我是对的！</p>
<p v-show="!ok">我是错的！</p>

3.2.4 v-model

  • v-model 指令用来在 input、select、text、CheckBox、radio 等表单控件上创建双向数据绑定。
  • 根据控件类型 v-model 自动选取正确的方法更新元素。
<div id="app">
  <form>
    姓名：
    <input type="text" v-model="data.name" placeholder="" />
    <br />

    性别：
    <input type="radio" id="man" value="One" name="sex" v-model="data.sex" />
    <label for="man">男</label>
    <input type="radio" id="female" value="Two" name="sex" v-model="data.sex" />
    <label for="female">女</label>
    <br />

    兴趣：
    <input type="checkbox" id="book" value="book" v-model="data.interests" />
    <label for="book">阅读</label>
    <input type="checkbox" id="swim" value="swim" v-model="data.interests" />
    <label for="swim">游泳</label>
    <input type="checkbox" id="game" value="game" v-model="data.interests" />
    <label for="game">游戏</label>
    <input type="checkbox" id="sing" value="sing" v-model="data.interests" />
    <label for="sing">唱歌</label>
    <br />

    身份：
    <select v-model="data.identity">
      <option value="teacher" selected>教师</option>
      <option value="worker">工人</option>
      <option value="doctor">医生</option>
    </select>
  </form>
</div>

<script>
  var vm = new Vue({
    el: '#app',
    data: {
      data: {
        name: '',
        sex: '',
        interests: [],
        identity: ''
      }
    }
  })
</script>
  • 除了上述用法，在 v-model 指令后还能添加多个参数
    • number：将用户的输入自动转换为 Number 类型（如果原值的转换结果为 NaN，则返回原值）
    • lazy：将数据改到在 change 事件中发生（当 input 输入框失去焦点的时候 msg 的值才发生变化）。
    • debounce：设置一个最小延时，在每次敲击后延时同步输入框的值与数据。(已经废弃，需要的话使用 lodash 的 debounce 方法来消抖)，参考https://segmentfault.com/a/1190000015312430 网址
<!-- lazy 参数 -->
<div id="app">
  <input v-model.lazy="msg" />
  <br />
  { {msg} }
</div>

<script type="text/javascript">
  var vm = new Vue({
    el: '#app',
    data: {
      msg: '内容是在 change 事件后才改变的'
    }
  })
</script>

3.2.5 v-for

  • 我们可以使用 v-for 指令基于原数据重复渲染元素。
<div id="app">
  <ul>
    <li v-for="(item,index) in items" :key="index">
      { {index} } -{ {item.parentMessage} } { {item.msg} }
    </li>
  </ul>
</div>

<script type="text/javascript">
  var vm = new Vue({
    el: '#app',
    data: {
      items: [
        { parentMessage: '滴滴' },
        { msg: '滴滴顺风车' },
        { msg: '滴滴专车' }
      ]
    }
  })
</script>
  • key 的作用：帮助 Vue 区分不同的元素，从而提高性能。

3.2.6 v-text

  • v-text 指令可以更新元素的 textContent。
<span v-text="msg"></span><br />
<!-- same as -->
<span>{ {msg} }</span>

3.2.7 v-html

  • v-html 指令可以更新元素的 innerHTML，不建议直接在网站上动态渲染任意 HTML 片段，容易导致 XSS 攻击。
<div v-html="html"></div>
<!-- same as -->
<div>{ {{html} }}</div>

3.2.8 v-bind

  • v-bind 指令用于响应更新 HTML 特性，将一个或多个 attribute，或者一个组件 prop 动态绑定到表达式
<!-- 绑定 attribute -->
<img v-bind:src="imageSrc" />
<!-- 缩写 -->
<img :src="imageSrc" />

<div id="app">
  <div :class="[classA,{classB:isB,classC:isC}]"></div>
</div>

<script type="text/javascript">
  var vm = new Vue({
    el: '#app',
    data: {
      classA: 'A',
      isB: false,
      isC: true
    }
  })
</script>

3.2.9 v-on

  • v-on 指令用于绑定事件监听器。
  • 使用在普通元素上，只能监听原生 DOM 事件；使用在自定义元素组件上，可以监听子组件触发的自定义事件
  • 如果在内联语句处理器中访问原生 DOM 事件，则可以用特殊变量 $event 把它传入方法。
  • v-on 后面不仅可以跟参数，还可以增加修饰符
    • .stop：停止冒泡
    • .prevent：停止默认行为
    • .self：只当事件是从侦听器绑定的元素本身触发时才出发回调
<!-- 方法处理器 -->
<button v-on:click="doThis"></button>
<!-- 内联语句 -->
<button v-on:click="doThat('hello',$event)"></button>
<!-- 缩写 -->
<button @click="doThis"></button>

<!-- 事件修饰符 -->
<!-- 停止冒泡 -->
<button @click.stop="doThis"></button>
<!-- 阻止默认行为 -->
<button @click.prevent="doThis"></button>
<!-- 串联修饰符 -->
<button @click.prevent.stop="doThis"></button>
<!-- 键修饰符，键别名 -->
<input @keyup.enter="onEnter" />
<!-- 键修饰符，键代码 -->
<input @keyup.13="onEnter" />
<!-- 自定义按键修饰符 全局config.keyCodes 对象 -->
Vue.config.keyCodes.aaa = 65

  • 事件绑定 – 参数传递

    • 如果事件直接绑定函数名称，那么默认会传递事件对象作为事件函数的第一个参数。
    • 如果事件绑定函数调用（内联语句的形式），那么事件对象必须作为最后一个参数显示传递，并且事件对象的名称必须是 $event
    <div id="app">
    <button v-on:click="handle1">点击1</button>
  <button v-on:click="handle2(123,"hello",$event)">点击2</button>
</div>

<script src="js/vue.js">
    var vm = new Vue({
    el:"#app",
    data:{

    },
    methods:{
      handle1: function (event) {
        console.log(event.target.innerHTML)
        // 点击1
      },
      handle2: function (p1,p2,event) {
        console.log(event.target.innerHTML)
        // 123 "hello" 点击二
      }
    }
  })
</script>

3.2.10 v-ref

  • 在父组件上注册一个子组件的索引，便于直接访问。
  • 不需要表达式，必须提供参数 id
  • 可以通过父组件的 $refs 对象访问子组件

3.2.11 v-slot

  • 提供具名插槽或需要接收 prop 的插槽

3.2.12 v-pre

  • 编译时跳过当前元素和其子元素。
  • 用来显示原始的标签

3.2.13 v-cloak

  • 和 CSS 规则如 [v-cloak]{display:none} 一起使用时，这个指令可以隐藏未编译的 双大括号标签直到实例准备完毕。
<style>
  [v-cloak] {
    display: none;
  }
</style>
<body>
  <div v-cloak>
    { {msg} }
  </div>
</body>

3.2.14 v-once

  • 只编译一次，显示后的内容不再具有响应式功能。

  • 元素/组件及其所有的子节点都将被视为静态资源内容并跳过

  • 目的是为了节省性能。

4.1 自定义属性

4.1.1 为何需要自定义指令

​ 内置指令不满足需求

4.1.2 自定义指令的语法规则

<input type="text" v-focus />

<script>
  Vue.directive('focus', {
    inserted: function (el) {
      // el 表示指令所绑定的元素
      el.focus()
    }
  })
</script>

4.1.3 带参数的自定义指令

<input type="text" v-color="msg" />

<script>
  Vue.directive('color', {
    bind: function (el, binding) {
      // el 表示指令所绑定的元素
      el.style.background = binding.value.color
    }
  })
  var vm = new Vue({
    data: {
      msg: {
        color: orange
      }
    }
  })
</script>

4.1.4 局部指令

// 局部指令只能在本组件使用
var vm = new Vue({
  data: {},
  directives: {
    color: {
      bind: function (el, binding) {
        el.style.background = binding.value.color
      }
    },
    focus: {
      inserted: function (el) {
        el.focus()
      }
    }
  }
})

5.1 计算属性

5.1.1 为何需要计算属性

​ 表达式的计算逻辑可能会比较复杂，使用计算属性可以使模板内容更加简洁

5.1.2 计算属性的语法规则

<div id="app">
  <div>{ {msg} }</div>
  <div>{ {reverseString} }</div>
</div>

<script>
  var vm = new Vue({
    data: {
      msg: 'nihao'
    },
    computed: {
      // 计算属性基于 data 中的值来处理
      reverseString: function () {
        return this.msg.split('').reverse().join('')
      }
    }
  })
</script>

5.1.3 计算属性和方法的区别

  • 计算属性是基于它们的依赖进行缓存的
  • 方法不存在缓存

6.1 过滤器

6.1.1 过滤器的作用

​ 格式化数据，比如将字符串格式化为首字母大写，将日期格式化为指定格式等。

6.1.2 自定义过滤器

Vue.filter('upper', function (value) {
  return value.charAt(0).toUpperCase() + value.slice(1)
})

6.1.3 过滤器的使用

<div>{ {msg | upper} }</div>
<div>{ {msg | upper | lower} }</div>
<div v-bind:id="id | formatId"></div>

6.1.4 局部过滤器

<script>
  var vm = new Vue({
    data: {
      msg: 'nihao'
    },
    // 只能在本组件中使用
    filters: {
      upper: function () {}
    }
  })
</script>

6.1.5 带参数的过滤器

<div>{ {date | format("yyyy-MM-dd")} }</div>

<script>
  var vm = new Vue({
    Vue.filter('format',function(value,arg) {
        var nowDate
          return nowDate += value.getFullYear() + "-" +(value.getMonth() + 1) + "-" + value.getDate()
      })
    data:{
      date:new Date()
    }
  })
</script>

7.1 侦听器

7.1.1 侦听器的触发条件和应用场景

  • 数据一旦变化就会通知侦听器所绑定方法
  • 当数据变化时执行异步操作或开销较大的操作用侦听器

7.1.2 侦听器的语法规则

<div id="app">
  <span>
    用户：
  </span>
  <input type="text" v-model.lazy="uname" />
  <span>{ {tip} }</span>
</div>

<script>
  var vm = new Vue({
    el: '#app',
    data: {
      uname: '',
      tip: ''
    },
    methods: {
      checkName: function (uname) {
        var that = this
        setTimeout(function () {
          if (uname == 'admin') {
            that.tip = '用户已存在，请更改用户名'
          } else {
            that.tip = '可以使用当前用户名'
          }
        }, 2000)
      }
    },
    watch: {
      uname: function (val) {
        this.checkName(val)
        this.tip = '正在验证...'
      }
    }
  })
</script>

10.1 数组更新检测

10.1.1 变更方法

​ 变更方法会变更调用了这些方法的原始数组，Vue 将被侦听的数组进行了包裹，所以它们也将会触发视图更新，这些被包裹的方法有：

  • push()
  • pop()
  • shift()
  • unshift()
  • splice()
  • sort()
  • reverse()

10.1.2 替换数组

​ 它们不会变更原始数组，而总是返回一个新数组。当使用非变更方法时，可以用新数组替换旧数组，这些方法有：

  • filter()
  • concat()
  • slice()

10.1.3 修改响应式数据

  • Vue.set(vm.items, indexOfItem, newValue)
  • vm.$set(vm.items, indexOfItem, newValue)
    • 第一个参数表示要处理的数组名称
    • 第二个参数表示要处理的数组的索引
    • 第三个参数表示要处理的数组的值

12.1 组件注册

### 12.1.1 全局组件注册
<div id="app">
  <!--驼峰式命名的组件，在使用的时候只能在标签模板中必须使用短横线的方式-->
  <button-counter></button-counter>
</div>

<script>
  // 可以使用驼峰式命名法和短横线命名 buttonCounter
  Vue.component('button-counter', {
    // data 必须是一个函数
    data: function () {
      return {
        count: 0
      }
    },
    // 组件模板必须只有一个根元素
    // 组件模板内容可以是模板字符串
    template: `<button @click="handle">点击了{ {count} }次</button>`,
    methods: {
      handle: function () {
        this.count++
      }
    }
  })
</script>

12.1.2 局部组件注册

<div id="app">
  <hello-world></hello-world>
</div>

<script>
  var HelloWorld = {
    data: function () {
      return {
        msg: 'HelloWorld'
      }
    },
    template: `<div>{ {msg} }</div>`
  }

  var vm = new Vue({
    el: '#app',
    data: {},
    components: {
      'hello-world': HelloWorld
    }
  })
</script>

13.1 组件间数据交互

13.1.1 父组件向子组件传值

<div id="app">
  <!-- 父组件通过属性来将值传递给子组件 -->
  <menu-item :title="title"></menu-item>
</div>

<!-- 组件内部通过 props 接收传递过来的值 -->
<script>
  Vue.component('menu-item', {
    props: ['title'],
    template: '<div>{ {title} }</div>'
  })
  var vm = new Vue({
    el: '#app',
    data: {
      title: '来自父组件的值'
    }
  })
</script>

【注意点】：

  • DOM 元素不区分大小写，所以在 props 中使用驼峰形式，模板中需要使用短横线的形式，但在字符串模板( template )中没这个限制。

  • props 属性值类型：

    • 字符串 String
    • 数值 Number
    • 布尔值 Boolean
    • 数组 Array
    • 对象 Object

    其中对于 数值 Number 和 布尔值 Boolean，在父组件中通过 v-bind 动态绑定数据，子组件获取得数据的数值类型为 Number 或 Boolean，否则都为 String。

13.1.2 子组件向父组件传值

  • 子组件通过自定义事件向父组件传递信息
<button v-on:click='$emit("enlarge-text",5)'>扩大字体</button>
  • 父组件监听子组件的事件
<menu-item v-on:enlagre-text="fontSize += $event"></menu-item>

【注意点】：props 是单向数据流，我们不应该通过 props 来从子组件向父组件传值。

13.1.3 兄弟组件间传值

// 单独的事件中心管理组件间的通信
var eventHub = new Vue()

// 监听事件
eventHub.$on('add-todo', function (val) {})

// 触发事件
eventHub.$emit('add-todo', val)

// 销毁事件
eventHub.$off('add-todo')

14.1 组件插槽

<div id="app">
  <alert-box>有个错误</alert-box> // ERROR:有个错误
  <alert-box>有个警告</alert-box> // ERROR:有个警告 <alert-box></alert-box> //
  ERROR:默认内容
</div>

<script>
  Vue.component('alert-box', {
    template: `
            <div>
              <strong>ERROR:</strong>
                <slot>默认内容</slot>
          </div>
        `
  })
</script>

14.1.1 插槽作用域

<!-- 父级插槽可以访问相同实例 property 作用域，即vm实例的data值，但不能访问 test-one 组件的作用域 -->
<!-- 插槽内容是传递给 test-one 的，而不是在 test-one 组件内部定义的 -->

<div id="app">
  <test-one>
    Logged in as { { user.name } }
    <!-- 显示结果为 saber -->
  </test-one>
</div>

<script>
  Vue.component('test-one', {
    template: `
     <a href="http://www.baidu.com">
       <slot></slot>
     </a>
      `,
    data: function () {
      return {
        user: {
          name: 'artoria'
        }
      }
    }
  })

  var vm = new Vue({
    el: '#app',
    data: {
      user: {
        name: 'saber'
      }
    }
  })
</script>

14.1.2 具名插槽

​ 有时我们需要多个插槽。如下例 组件：

<div class="container">
  <header>
    <!-- 我们希望把页头放这里 -->
  </header>
  <main>
    <!-- 我们希望把主要内容放这里 -->
  </main>
  <footer>
    <!-- 我们希望把页脚放这里 -->
  </footer>
</div>

​ 我们可以通过 元素的 name 属性来具体名称的插槽

<div class="container">
  <header>
    <slot name="header"></slot>
  </header>
  <main>
    <slot></slot>
  </main>
  <footer>
    <slot name="footer"></slot>
  </footer>
</div>

​ 在向具名插槽提供内容的时候，我们需要在 < template > 元素上使用 v-slot 指令。

<base-layout>
  <template v-slot:header>
    <h1>Here might be a page title</h1>
  </template>

  < template >
    <p>A paragraph for the main content.</p>
    <p>And another one.</p>
  </template>

  <template v-slot:footer>
    <p>Here's some contact info</p>
  </template>
</base-layout>

​ 【注意点】：v-slot 只能添加在 < template > 上。

14.1.3 作用域插槽

​ 有时我们需要让插槽的内容能够访问子组件中的数据。

<current-user>
  <!-- 这里的插槽内容只能获取本实例即 vm 的 data 数据，而获取不到子组件的数据 -->
  { {user.firstName} }
</current-user>

<script type="text/javascript">
  Vue.component('current-user', {
    data: function () {
      return {
        user: {
          firstName: 'Artoria',
          lastName: 'Pendragon'
        }
      }
    },
    template: `
        <span>
          <slot>
            { {user.lastName} }
          </slot>  
        </span>
      `
  })
</script>

​ 我们可以通过作用域插槽来解决这个问题。

<!-- 在子组件的模板中 -->
<!-- 绑定在 <slot> 元素上的 attribute 被称为 插槽 prop -->
<span>
  <slot :user="user">
    { {user.lastName} }
  </slot>
</span>

<!-- 在父级作用域中，我们可以使用带值的 v-slot 来定义我们提供的插槽的名字 -->
<current-user>
  <template v-slot="slotProps">
    { {slotProps.user.firstName} }
  </template>
</current-user>

【注意点】：

  • 当被提供的内容只有默认插槽的时候，组件的标签可以被当成模板来使用，这时我们可以把 v-slot 直接用在组件上。
  • 默认插槽的缩写语法不能和具名插槽混用，会导致作用域不明确
  • 可以使用 解构解析 来传入具体的 插槽 prop，如上例：
<current-user>
  <template v-slot="{ { user } }">
    { {user.firstName} }
  </template>
</current-user>
